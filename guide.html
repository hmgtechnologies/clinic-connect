Clinic Connect — Operator & Deployment Guide

Everything you need to build an enterprise-grade clinic platform and deploy it — using only free tools (Supabase free tier + free static hosting). No AI API — it is not cost effective.

Two parts

(A) Use the builder to create the platform. (B) Follow deployment to make it live for the clinic.

A · Using the Builder

  1. Open builder.html.
  2. Step 1 — Clinic Details: name, motto, contact, bank/payment details, currency.
  3. Step 2 — Branding: upload the clinic logo, pick a colour theme and a font.
  4. Step 3 — Layout & UI: choose Sidebar, Top‑nav or Card‑hub navigation.
  5. Step 4 — Structure: tick the clinic's departments, age groups & activities.
  6. Step 5 — Features: toggle modules (members & dashboard are always on).
  7. Step 6 — Connect & Generate: optionally paste Supabase keys, then either Preview Platform (click through every page with sample data) or Generate & Download the ZIP.

Save / Load: use the Save button to keep a client's answers as a JSON file, and Load to bring them back later. Your work also auto‑saves in the browser.

B · Deploying the Generated Platform

Inside the downloaded ZIP you'll find a README.md with these same steps.

Step 1 — Create the free database (Supabase)

  1. Go to supabase.com → create a free account → New project (save the database password).
  2. Wait ~2 minutes for it to provision.
  3. Open SQL Editor → New query, paste ALL of database/schema.sql from the ZIP, click Run. (Safe to re‑run.)
  4. (Optional, quicker logins) Authentication → Providers → Email → turn Confirm email OFF.

Step 2 — Connect the app to the database

  1. In Supabase: Project Settings → API. Copy the Project URL and the anon public key.
  2. Open assets/js/config.js in the ZIP and paste them into SUPABASE_URL and SUPABASE_KEY.
  3. 🔒 Never paste the service_role key here — only the anon key belongs in the browser.

Step 3 — Publish online (pick one, all free)

  • GitHub Pages: create a repo → upload the ZIP's contents (so index.html is at the root) → Settings → Pages → Deploy from branch → main → / (root).
  • Cloudflare Pages / Vercel: import the repo or drag the folder → Framework: None → output dir: / → Deploy.

Step 4 — Create the first admin

  1. Open the live site → Request Access → sign up.
  2. In Supabase → SQL Editor, run (with your email):
UPDATE profiles SET role='admin', status='approved'
WHERE email='you@example.com';

Refresh — admin tools unlock. New sign‑ups stay pending until an admin approves them.

Admin Capabilities (oversight & coordination)

  • People & Roles: approve sign‑ups and set each person's role — staff, manager or admin — on the People page.
  • CRM / Customers: capture leads & customers and move them through a pipeline (lead → contacted → qualified → customer).
  • Sales / Deals: track opportunities and revenue by stage.
  • Invoices: create invoices, track paid/outstanding, and print a branded invoice PDF per customer.
  • Inventory: manage stock with quick +/- adjustments and low‑stock alerts.
  • Finance: income & expense ledger with live balance.
  • Projects, Tasks, Appointments, Support Tickets, HR/Leave: run day‑to‑day operations.
  • Broadcast: notify staff or customers in bulk via WhatsApp/Email/SMS.
  • Events / Announcements / Documents / Gallery / Feedback: communicate, share & collect reviews.
  • Departments & Units: see headcount per department for coordination.

Enterprise Modules (new in v2)

All built on the same free Supabase + static stack — no extra cost, no AI APIs.

  • Point of Sale (POS): a fast retail checkout. Tap products into a cart and complete a sale — it records an order, posts the income to Finance, and automatically reduces Inventory.
  • Online Orders / Storefront: a public shop.html page (no login) where customers browse in‑stock products and place orders. Orders appear in the admin Online Orders page with status control.
  • Quotations / Estimates: create branded quotes, print them as PDF, and convert an accepted quote into an invoice with one click.
  • Purchase Orders & Suppliers/Vendors: raise POs to suppliers, track receiving, and keep a categorised supplier directory.
  • Asset Register: track equipment — tag, category, location, who holds it, value and status (in‑use / repair / retired).
  • Expense Claims: staff submit expenses; managers approve/reject. Approved claims auto‑post to Finance as expenses.
  • Payroll & Payslips: run a salary (gross − deductions = net), auto‑post the cost to Finance, and print a payslip PDF per staff.
  • Timesheets: staff log hours per day/project for billing & payroll.
  • Goals / KPIs / Appraisal: set objectives with targets and visual progress bars; update progress as work is done.
  • Recruitment: track job applicants through a hiring pipeline.
  • Contracts & Renewals: store agreements with value & expiry — items expiring within 30 days are flagged for renewal.
  • Knowledge Base / SOPs: a searchable library of policies & how‑tos.
  • Compliance & Policies: a register of policies with review dates and compliance status.
  • In‑App Inbox: private staff↔management messaging inside the app.
  • Reports & Export: download any dataset (staff, sales, finance, etc.) as Excel (.xlsx) or CSV for accounting, audits & backup.
  • Analytics & Charts: visual dashboards — income vs expense, sales pipeline and a monthly income trend (Chart.js, loaded free via CDN).
  • Audit Log: an automatic trail of key admin actions (approvals, role changes, sales, payroll, etc.) for accountability & oversight.
  • Settings & Backup: view the org profile, list active modules, and download a full JSON data backup of the entire database anytime.

Point of Sale (POS) — receipts & barcode scanning

  • Scan to sell: a USB barcode scanner types the SKU and adds the product automatically (Enter), or tap Camera to scan with a phone/tablet camera (uses the free html5-qrcode library via CDN).
  • Discounts & change: apply a discount, choose a payment method, enter the amount tendered and the screen shows the change due.
  • Receipts: completing a sale opens a printable, thermal-style receipt and (as before) records the order, posts income to Finance and reduces Inventory automatically.
  • Tip: give each product a SKU/barcode and a reorder level in Inventory so scanning and low-stock alerts work well.

Free Automation Pack (optional)

If included, the ZIP contains supabase/functions/ with ready-made Edge Functions and a step-by-step docs/AUTOMATION.md. These are 100% free (Supabase free tier + Resend free email tier) and use NO paid AI API. The app works fully without them.

  • daily-digest — posts a daily summary (income, expense, orders, low stock) to the in-app Inbox and emails admins.
  • low-stock-alert — emails admins + Inbox notice when stock hits the reorder level.
  • renewal-reminders — flags expiring contracts and unpaid invoices.

Deploy with the free Supabase CLI and schedule with free Supabase Cron — full commands are in docs/AUTOMATION.md. The service_role key stays only in Supabase's secret vault, never in your website code.

Security model (Row Level Security)

  • Every table has RLS on. Signed‑in staff can read shared data; only admins can write management tables.
  • Expenses, timesheets & leave use per‑user rules: each person can only submit/see their own rows; admins manage all.
  • The public storefront can insert orders only (it cannot read or edit any other data) — safe to share publicly.
  • The service_role key is never used in the browser; only the anon key goes in config.js.

Roles Explained

  • admin — full access to everything (oversight & management).
  • manager — department/team lead working across modules.
  • staff — team member; submits leave, tasks, tickets, etc.

Multi-branch

If you list branches/locations in Step 1, a branch selector appears in the platform header so larger clinics/hospitals can organise by location.

Troubleshooting

  • "supabase is not defined": keep the script tags in index.html in order (library before config.js).
  • Nothing loads: re‑check the URL/anon key in config.js.
  • Can't see admin tools: run the admin SQL in Step 4.
  • Photos won't upload: ensure you ran the full schema.sql (it creates the storage buckets).
